# v0.1.0

## Highlights
- Rust TUI to navigate a curated automation workspace and run scripts with guided prompts.
- JSON schema-driven parameter collection for scripts that describe their inputs.
- Workspace layout with curated flavors (`.omaken`), execution history, and optional folder widgets.

## Added
- Cross-platform CLI commands: `update`, `uninstall`, `doctor`/`check`, `scripts`, `run`, `init`,
  `config`, `env`, `list`, `install`, and `completion`.
- Script discovery and execution for `.bash`, `.sh`, `.ps1`, and `.py` under
  `~/Documents/omakure-scripts` (Windows: `%USERPROFILE%\\Documents\\omakure-scripts`).
- TUI navigation with folder widgets via `index.lua` to render custom folder screens.
- Execution history saved in `.history/` for each script run.
- Script templates with `omakure init` (extension-based: `.bash`, `.sh`, `.ps1`, `.py`).
- Installers for Linux/macOS (curl | bash) and Windows (PowerShell).
- Update flow with optional version pinning and repo override; syncs new scripts without
  overwriting existing ones.
- Uninstall flow with optional `--scripts` to remove the workspace folder.
- Shell completion generation for bash, zsh, fish, and PowerShell.
- Environment overrides: `SCHEMA_MODE=1` for schema output and `OMAKURE_SCRIPTS_DIR` to
  customize the workspace location.
- Example Azure scripts (`rg-list-all`, `rg-details`, `rg-delete`) plus an example Lua widget.
